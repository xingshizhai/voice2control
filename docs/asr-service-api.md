# 本地 ASR 服务 API 文档

> 直接调用局域网内 ASR 服务的 WebSocket API，无需额外封装层。

---

## 服务说明

部署在局域网内的 **ASR 语音识别服务**提供原生 WebSocket API，任何程序均可直接连接调用。

---

## 接入端点

| 端点 | 协议 | 适用场景 | 推荐 |
|------|------|---------|------|
| **ws://192.168.1.88:6006**（示例） | WS（明文） | 程序接入（内网） | 推荐 |
| **wss://192.168.1.88:6443**（示例） | WSS（加密） | Web / HTTPS 页面 | Web 使用 |

将示例 IP 换成你局域网内 ASR 服务所在主机的地址。

---

## 协议说明

### 请求

**格式**：Binary Message  
**内容**：完整 WAV 文件（含 WAV 头）  
**要求**：

- 采样率：16000 Hz  
- 声道：1（单声道）  
- 位深：16-bit PCM  

### 响应

**格式**：JSON  

```json
成功:
{
    "text": "识别的文本",
    "status": "success"
}

失败:
{
    "error": "错误信息",
    "status": "error"
}
```

---

## 音频格式要求

| 参数 | 要求 |
|------|------|
| 格式 | WAV（标准头） |
| 采样率 | 16000 Hz |
| 声道 | 1 |
| 位深 | 16-bit |

---

## 客户端实现要点（与本仓库一致）

- 将采集到的 **s16le PCM** 封装为 **WAV** 后，以 **单次二进制帧** 发送。  
- 使用 **`ws://`** 时无需 TLS；使用 **`wss://`** 时若服务为自签名证书，客户端需配置「不校验证书」或导入 CA（仅建议在可信内网使用）。  
- Python 实现见 `vc/asr.py`（依赖 `websocket-client`）。

---

## 故障排查

| 问题 | 处理 |
|------|------|
| 连接被拒绝 | 检查 ASR 主机网络、防火墙、服务进程与端口（如 6006 / 6443） |
| 识别结果为空 | 检查 WAV 格式、音量、音频是否含有效语音 |
| 超时 | 检查网络负载、适当增大客户端超时 |
